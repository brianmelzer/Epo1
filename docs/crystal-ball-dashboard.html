How this dashboard is built

One card per Dashboard section: what it shows, where the data comes from, and exactly how each number is computed. Nothing on Tab 1 is hand-entered — every figure is either a ledger aggregate or a live SQL result.

§KPI strip

Source

data/callout_ledger.csv — the scored call-out ledger (34 call-outs), refreshed by sql/06_ledger_refresh.sql against shaundatabase.v_so_history (internal sell-through).

Each stat, defined

  • Call-outs scored — count of ledger rows with a computed IQS.
  • Mean IQS — simple average of the iqs column across scored rows.
  • % matched — share of call-outs whose keywords matched at least one internal item with sell-through.
  • Reorder rate — share of rows with reordered = yes.
  • Total realized gross — sum of realized_gross across rows. Note: paired call-outs (e.g. cast-aluminum cookware, candle warmer, beverage dispenser) describe the same concept twice, so gross is attributed per call-out and double-counts some concepts.
  • Last scored_asof — max of the scored_asof column = date of the last flywheel refresh.

The IQS formula (F13 Item Quality Score)

IQS = 0.20·min(items/3, 1) + 0.35·min(margin/target, 1) + 0.25·min(units/50000, 1) + 0.20·min(active_months/12, 1)

Computed per call-out by sql/06_ledger_refresh.sql over matched items in shaundatabase.v_so_history. Outcome windows are set per row: 2025 seed call-outs are scored on a different realization window than 2026 picks, so early picks aren't penalized for having had less time to sell.

§The two-loop visual

Inner loop — the signal engine

Runs over the crystalball schema: 169M+ daily Amazon rank rows since 2021-08, plus Walmart, Five Below, weekly search terms, and social signals. It measures, forecasts, reconciles against Epoca POS, and recommends.

Outer loop — the commercial flywheel

The real business journey: call-out → pitch → MRF → PO → ship → sell-through → reorder. Sourced from Librarian call-out documents plus sqlmas90 (invoices, POs) and shaundatabase (inventory, sell-through).

The coupling

Recommend feeds Call-out (inner → outer), and commercial outcomes (IQS) retrain the inner loop's weights (outer → inner). That feedback is the flywheel.

Where the node numbers come from

  • Rank-row scale — live row count of crystalball.list_item.
  • Gross, reorder rate, IQS, call-outs scored — aggregates of the call-out ledger (same CSV as the KPI strip).

§IQS leaderboard

Source & sort

The ledger CSV, sorted by IQS descending. Each row is one call-out graded on its real outer-loop journey.

How call-outs are matched to sales

Matching is synonym-aware regex against internal item descriptions — e.g. CA = cast aluminum, CER = ceramic — using each row's match_keywords patterns.

Caveats

  • 2 rows found zero matches (honest zeros, shown as "no match" on Tab 1).
  • Broad patterns can over-match, attributing unrelated items' sales to a call-out.
  • The precise fix is curation: populating linked_item_codes per call-out replaces regex guessing with exact item attribution.

§Signals now

Source

Live SQL against crystalball.list_item (Amazon daily rankings) using the sql/01_breakout_radar.sql pattern.

Velocity (log-rank momentum, F1)

velocity = (−ln(avg rank, last 14d)) − (−ln(avg rank, 30–44d prior))

Lower rank = more popular, so a positive velocity means the item is climbing. The log makes a 5→3 move count more than 80→78.

Review gain (sales proxy)

review_gain = max(review_count, last 7d) − max(review_count, 23–37d prior)

New reviews trail sales, so review gain confirms that rank velocity is demand, not noise.

Data-as-of stamp

max(cb_stamp) — the newest observation date in the table at query time.

Honesty footer.
